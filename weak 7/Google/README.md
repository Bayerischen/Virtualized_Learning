# Sprint

程序主要代码很简短，一个是输入，一个mmap，一个sprintf，但是其中有很多的格式化字符串，最开始看这么多的格式化字符串的时候是有点懵的，然后仔细的看了一看，他是利用了格式化字符串的某些不常用的功能，可以更改参数的值，并且可以让参数进行加减操作，其实这时候就可以看出来了，参数其实相当于寄存器的功能，也就是说这是个通过格式化字符串弄的虚拟机，sprintf就是启动虚拟机

然后来分析格式化字符串，格式化字符串对应有几种特殊的情况，包括边界问题，条件分支，现再随便用几个字符串举个例子

```
%1$00038s%3$hn%1$65498s%1$28672s%9$hn
```

1. `%1$00038s%3$hn`：这个的意思是把38写入到第3个参数
2. `%1$65498s`：这个是让38+65498，但是这里是有一个边界问题的，可以发现00038和65498都是5位的，然后把38和65498转换成二进制数，对应就是`0000000000100110`和`1111111111011010`，可以看到如果说是范围就是16位数的二进制数那么这两个的二进制数相加之后应该是0，所以说就可以发现中间的这部分就是清0
3. `%1$28672s%9$hn`：这个和第一个是一样的道理

```
%1$00074s%3$hn%1$65462s%1$*8$s%7$hn
```

1. `%1$*8$s%7$hn`：只有这里和上面是有些出入的所以指列出来这一个了，其中`*8$s`这个的意思是取第8个参数的值，所以说这句的意思就是把第8个参数的值写入到第7个参数中

```
%1$00149s%3$hn%1$65387s%1$*8$s%1$2s%7$hn
%1$00264s%3$hn%1$65272s%1$*10$s%1$*10$s%17$hn
%1$00310s%3$hn%1$65226s%1$28672s%1$*16$s%7$hn
```

1. `%1$*8$s%1$2s%7$hn`：这种情况也不算是特殊情况，主要是有一个相加的功能，也就是第8个参数的值加上2然后赋值给第7个参数
2. `%1$*10$s%1$*10$s%17$hn`：其实这个也没什么变化就是相加的两个东西变了，这里是第10个参数加上第10个参数然后赋值给第17个参数
3. `%1$28672s%1$*16$s%7$hn`：这里和第一个一样只是加的顺序变化了，上面是参数加数字，这里是数字加参数

```
%1$00468s%3$hn%1$65068s%1$65519s%7$hn
```

1. `%1$65519s%7$hn`：这里涉及了边界问题，65519的二进制数是`1111111111101111`，对应16位的有符号数，第一位为1代表说这是一个负数，所以在我们转换伪代码的时候需要把他转换成负数，65519对应的负数也就是求补码，`1000000000010001`对应的就是-17

```
%14$c%1$00186s%2$c%4$s%1$00429s%3$hn
```

1. **这个我觉得是最巧妙的地方，把格式化字符换的功能和sprintf函数的功能很好的结合在了一起，起到了一个if判断的功能**
2. **`sprintf`：这个函数可以有多个参数，其中第一个参数是一个buffer，第二个参数是格式化字符串，第三个参数及之后的参数都是对应的格式化字符串中的参数，sprintf有一个功能就是会把输出的字符存储到buffer里面**
3. **把这句拆分来看**
   - **`%14$c`：这个是取第14个参数，但是只取一个字符**
   - **`%1$00186s`：这个还是写186个字符**
   - **`%2$c`：取第2个参数，其中第2个参数是'\x00'，这里sprintf会遇到'\x00'截断，也就是说遇到'\x00'就不会再存到buffer里了**
   - **`%4$s`：在题中第4个参数对应的是存储字符的buffer，所以这里就是会加上buffer里面的字符数，这里就会有一个分支了，如果说第14个参数的值是会影响buffer的内容的，如果说从第14个参数取的字符是'\x00'，那就代表说buffer会直接被截断，buffer的字符数量就是0，如果说第14个参数取的字符不是'\x00'，那buffer就不会被截断，buffer的值就是1+186+1+187+429（`%14$c`+`%1$00186s`+`%2$c`+`%14$c`+`%1$00186s`+`%1$00429s%3$hn`）**
